# Ресурсный план — краткая спецификация

## Назначение

Проект предназначен для быстрого формирования ресурсного плана команды на основе графика отпусков, настроек ролей и выбранного периода. Система помогает оценить доступность сотрудников, рассчитать нагрузку по ролям и сформировать итоговый Excel-файл для планирования.

## Что делает система

Пользователь проходит мастер из нескольких шагов:

1. Загружает Excel-файл графика отпусков.
2. Выбирает команду.
3. Отмечает подразделения и сотрудников.
4. Выбирает период планирования.
5. Запускает анализ.
6. Скачивает итоговый ресурсный план в Excel.

Дополнительно система позволяет:

- сохранять и переиспользовать состав команд;
- настраивать роли, виды работ и часы;
- задавать распределение времени команды;
- учитывать производственный календарь;
- пересчитывать рабочие дни с учетом отпусков и процента участия сотрудника.

## Интерфейс

Интерфейс построен как пошаговый мастер с современным светлым визуальным стилем:

- верхняя панель управления;
- центральная колонка с 7 этапами;
- правый блок прогресса;
- боковая панель настроек;
- модальные окна для служебных сообщений.

Основной акцент сделан на простом сценарии: загрузка данных, выбор периода, анализ, выгрузка результата.

## Текущая архитектура

Текущая версия проекта реализована как монолитное веб-приложение:

- сервер отдает интерфейс и API;
- frontend работает через REST-запросы;
- бизнес-логика расчета вынесена в отдельные сервисы;
- настройки и импортированные данные сохраняются в JSON;
- итоговый Excel-файл формируется на сервере по шаблону.

## Основные API-методы

Система использует следующие ключевые методы:

- `GET /api/bootstrap`
- `POST /api/upload-vacations`
- `PUT /api/settings`
- `POST /api/reset-workflow`
- `GET /api/calendar/month-workdays`
- `POST /api/analyze`
- `POST /api/team-members/save`
- `POST /api/team-members/clear`
- `POST /api/team-members/refresh`
- `POST /api/export`

## Что учитывает расчет

При формировании плана система учитывает:

- рабочие дни периода;
- отпуска сотрудников;
- процент участия сотрудника;
- выбранные месяцы;
- принадлежность сотрудника к роли;
- режим группировки отчета.

Результатом становятся:

- оценка доступности команды;
- расчет часов по ролям и спринтам;
- агрегированные показатели по периоду;
- готовая Excel-выгрузка.

## Текущее состояние

Проект рабочий и запускается как Node.js-приложение на локальном сервере. Подготовлено окружение для дальнейшей миграции на новый стек, но основная логика пока остается в текущей реализации.

- упростить дальнейшее развитие и поддержку системы.

## Краткий вывод

Это внутренний инструмент ресурсного планирования, который автоматизирует работу с отпускными графиками, ролями и периодами планирования, уменьшает ручной расчет и ускоряет подготовку итогового Excel-плана.
