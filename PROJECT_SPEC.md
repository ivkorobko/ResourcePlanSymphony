# Спецификация проекта

## 1. Название

Ресурсный план (`ResPlan`)

## 2. Назначение

Система предназначена для пошагового формирования ресурсного плана команды на основании:

- Excel-файла графика отпусков;
- сохраненного состава команд;
- настроек ролей, категорий работ и распределения времени;
- производственного календаря;
- Excel-шаблона итоговой выгрузки.

Основной пользовательский результат:

- загрузить файл отпусков;
- выбрать команду, подразделения, сотрудников и период;
- получить расчет по рабочим неделям и ролям;
- сформировать итоговый Excel-файл ресурсного плана.

## 3. Бизнес-функции

Проект поддерживает следующие сценарии:

- импорт графика отпусков из исходного Excel-файла;
- сохранение и повторное использование состава команды;
- настройку ролей, групп ролей и категорий работ;
- настройку распределения времени команды;
- расчет рабочих дней по внешнему календарному API с fallback по будням;
- анализ нагрузки по выбранному периоду;
- экспорт итогового ресурсного плана в Excel.

## 4. Текущий стек

### 4.1. Backend

- Node.js
- Express 5
- CommonJS
- Multer для загрузки файлов

### 4.2. Frontend

- HTML
- CSS
- Vanilla JavaScript

### 4.3. Работа с Excel

- `exceljs`
- `xlsx-populate`

### 4.4. Хранение данных

- JSON-файлы в каталоге `data`
- база данных в текущей реализации не используется

### 4.5. Тестирование

- встроенный `node:test`

## 5. Целевой стек миграции

Подготовлен целевой стек для дальнейшей миграции:

- PHP 8.3
- Symfony 7.x
- PostgreSQL 17
- Symfony UX для UI-слоя

На текущем этапе этот стек подготовлен как окружение, но основная бизнес-логика проекта все еще работает на Node.js.

## 6. Архитектура текущей реализации

Приложение реализовано как монолит:

- Express отдает статические frontend-файлы из `public`;
- frontend работает через REST API;
- вычислительная логика вынесена в сервисы `src/services`;
- данные состояния сохраняются на диске в JSON;
- Excel-экспорт формируется на сервере по шаблону `Ресурсный план.xlsx`.

## 7. Структура проекта

Ключевые файлы и каталоги:

- `src/server.js` — HTTP-сервер и маршруты API
- `src/services/bootstrap.js` — инициализация данных, парсинг отпусков, дефолтные настройки
- `src/services/calendar.js` — получение рабочих дней по календарному API
- `src/services/plan.js` — расчет ресурсного плана
- `src/services/export.js` — генерация итогового Excel-файла
- `src/services/storage.js` — чтение и запись JSON-данных
- `src/utils/date.js` — утилиты дат
- `public/index.html` — разметка интерфейса
- `public/styles.css` — стили интерфейса
- `public/app.js` — клиентская логика
- `data/settings.json` — настройки проекта
- `data/vacations.json` — импортированные отпуска
- `test/*.test.js` — автотесты
- `start.bat` — основной файл запуска проекта на Windows
- `stop.bat` — остановка проекта на порту `3003`

## 8. Пользовательский интерфейс

Интерфейс построен как мастер из 7 шагов:

1. Загрузка файла
2. Сохранение данных
3. Подразделения
4. Сотрудники
5. Период
6. Анализ
7. Ресурсный план

Дополнительно интерфейс содержит:

- правый степпер прогресса;
- боковой drawer настроек;
- вкладки ролей, календаря и прочих настроек;
- модальные окна для служебных сообщений.

Визуальный стиль:

- светлый стеклянный интерфейс;
- плавные тени и полупрозрачные поверхности;
- крупные аккордеоны этапов;
- карточки, плитки и chip-элементы для выбора.

## 9. Источники данных

### 9.1. Файл отпусков

Из Excel-файла извлекаются:

- подразделение;
- ФИО сотрудника;
- должность;
- периоды отпусков;
- суммарное количество отпускных дней.

После загрузки данные нормализуются и сохраняются в `data/vacations.json`.

### 9.2. Настройки

Настройки включают:

- роли;
- группы ролей;
- команды;
- состав команд;
- распределение времени;
- параметры календарного API;
- режим формирования отчета.

Они сохраняются в `data/settings.json`.

## 10. API

Текущий backend предоставляет следующие endpoints:

- `GET /api/bootstrap`
- `POST /api/upload-vacations`
- `PUT /api/settings`
- `POST /api/reset-workflow`
- `GET /api/calendar/month-workdays`
- `POST /api/analyze`
- `POST /api/team-members/save`
- `POST /api/team-members/clear`
- `POST /api/team-members/refresh`
- `POST /api/export`

## 11. Расчетная логика

Расчет строится на основе:

- выбранных месяцев;
- рабочих дней в периоде;
- отпусков сотрудников;
- процента участия сотрудника;
- роли сотрудника;
- настройки группировки отчета.

Система вычисляет:

- список рабочих недель/спринтов;
- доступность сотрудника по неделям;
- суммарные часы по ролям;
- агрегированные показатели по периоду;
- данные для Excel-экспорта.

## 12. Запуск проекта

Текущий основной способ запуска на Windows:

- `start.bat` — старт или перезапуск проекта на `http://127.0.0.1:3003`
- `stop.bat` — остановка процесса на порту `3003`

Node-скрипты:

- `npm start`
- `npm run dev`
- `npm test`

## 13. Текущее состояние миграции

На данный момент:

- основная бизнес-логика работает на Node.js;
- интерфейс работает как статический frontend;
- локально подготовлены portable `PHP` и `Composer`;
- локально поднят `PostgreSQL` в Docker;
- миграция на Symfony/PostgreSQL еще не завершена.

## 14. Ограничения

- проект зависит от структуры входного Excel-файла;
- часть расчетов опирается на внешний производственный календарь;
- при недоступности календарного API используется fallback-логика;
- текущая версия не использует БД как основное хранилище;
- целевой Symfony-стек пока не введен в эксплуатацию.
