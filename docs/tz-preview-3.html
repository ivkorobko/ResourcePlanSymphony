Техническое задание (ТЗ)

4. Нефункциональные требования

  • Время ответа API (кроме тяжелых операций) — до 3 секунд при типовой нагрузке.
  • Работа приложения при временной недоступности PostgreSQL.
  • Логи ошибок и запуска в директории logs/.

5. Требования к данным

  • Основное хранение: PostgreSQL таблица app_document.
  • Fallback: .symfony_tmp/var/state/*.json.
  • Демо-артефакты: Test_demo.xlsx и шаблон с одной записью.

6. API (минимум)

  • GET /api/bootstrap
  • POST /api/upload-vacations
  • POST /api/settings
  • GET /api/calendar/month-workdays
  • POST /api/analyze
  • POST /api/export
  • POST /api/reset-workflow

7. Критерии приемки

  1. Приложение стартует на http://127.0.0.1:3003.
  2. Загрузка файла отпусков работает.
  3. Настройки ролей/команд сохраняются.
  4. В разделе «Разное» доступно скачивание образца.
  5. Анализ и экспорт выполняются без критических ошибок.
